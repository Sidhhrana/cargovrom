OVERVIEW
This project started as a request for a multiplayer browser game with a cyberpunk drifting 3D car vibe, inspired by slowroads.io. The project evolved through several major rewrites because early versions were visually rough, too blocky, too laggy, or had broken world/track presentation. The current version is a lightweight Three.js browser racing prototype with procedural tracks, AI opponents, lap/position tracking, selectable difficulty, a race menu, a minimap, and a live dashboard.

Goal:

Implementation:

- Added Flow / Pro / Expert difficulty radio controls on home screen.
- Difficulty now affects:
  - track turn count
  - track radius / length
  - radius jitter
  - corner wave severity
  - point noise
  - AI skill
  - AI aggression
  - AI top speed
  - AI corner speed

Difficulty settings in current code:
Flow:
  turns: 8
  smoother radius/noise
  lower AI skill/aggression
  lower AI top speed/corner speed

Pro:
  more turns
  longer circuit
  more technical corners
  faster and more aggressive AI

Expert:
  most turns
  longest circuit
  strongest wave/noise values for harder turns
  fastest, most competitive AI

Terrain:

KNOWN LIMITATIONS / NEXT IMPROVEMENTS
1. True multiplayer is not implemented in current cleaned-up version.
Earlier versions had local tab presence, but the current race rewrite focuses on single-player AI racing.

2. Car model is intentionally simple.
This was a performance/stability decision after the GLB model caused lag and poor aesthetics.

3. Track generation is procedural but still abstract.
It creates varied fun circuits, not simulation-grade terrain engineering.

4. The dashboard is functional but can still be visually refined.
Needles work, but a more polished gauge face with tick marks would improve the iOS/racing feel.

5. AI is competitive but not physically simulated.
AI follows race distance/racing-line logic rather than running full vehicle physics.

6. The old GLB and loader files are still present.
They can be deleted later if you want to keep the repo lean.
